# AI向け生成フック（Config構築ルール）

コード・要件からConfigのJSONを自動生成する際のルールを以下に示す。

## 7.1 シート名変数の命名規則

`screen_layouts`でシート名を指定する。命名規則:

| 種別 | パターン | 例 |
|------|---------|-----|
| 機能レベル | `外部-画面レイアウト-{機能名}` | `外部-画面レイアウト-社員検索` |
| UIコンポーネント | `外部-画面レイアウト-{UI部品名称}` | `外部-画面レイアウト-現職エリア` |
| モーダル | `外部-画面レイアウト-{モーダル名}` | `外部-画面レイアウト-経歴モーダル` |

テンプレートの既存シート名（`template_sheet`）と出力名（`output_name`）を両方指定する。

## 7.2 外部-機能概要の生成ルール

`overview`セクションの3項目をコード・APIから以下のように導く:

### `description`（H9: 機能全体説明文）
**生成方法**: 以下の要素を統合して2〜4文で書く
- APIの名称から「何をする機能か」を推測
- リクエストパラメーターから「どんな条件で動くか」を読み取る
- レスポンスから「何を画面に反映するか」を読み取る
- 複数の表示モード（ブロック/一覧等）があれば、切替条件を`・`で列挙する

**書式テンプレート**:
```
{機能の目的}。
・{モード/条件1}の場合、{動作1}
・{モード/条件2}の場合、{動作2}
```

### `screens[].descriptions`（H列: 各コンポーネントの動作説明）
**生成方法**:
- 各APIのレスポンスマッピングから「何を表示するか」を1文で要約
- 表示モード（ブロック/一覧）がある場合はモードごとに行を分ける
- AA列の※条件（フラグ制御等）がある場合は重要なもの1件を添える

**書式テンプレート**:
```
{モード}の場合、{データ取得先}を取得して、{表示対象}を表示する。{主要制御条件}を制御する
```

### `flow_description`（H19: 処理フロー説明）
**生成方法**:
- APIコールの実行順序を`1.`から箇条書きで記述
- 呼出元からのデータ受け渡し（リクエストパラメーターの出所）を明記
- 分岐がある場合は条件を記載

**書式テンプレート**:
```
・{フロー全体の見出し}の流れを以下に示す。
1. {ステップ1（入力受取）}
2. {ステップ2（API呼び出し）}
3. {ステップ3（画面反映/分岐）}
```

## 7.3 processing.apis の生成ルール

### `notes`（※注記行）の生成基準
以下の条件のいずれかに該当する場合は**必ず**`notes`に追記する:

| 条件 | 書式 |
|------|------|
| レスポンスパラメーターにフラグがある | `※{パラメーター名}が{値}の場合、{動作（表示しない等）}` |
| null/none チェックが必要 | `※{パラメーター名}がnullの場合、{デフォルト動作}` |
| データなし時のデフォルト動作 | `※{対象}がない場合、{代替動作}を表示する` |
| 連動する画面要素への副作用 | `※{要素}が{状態}の場合、{他要素}へ影響する` |

**注記の優先順位**:
1. 機能利用フラグ/表示フラグ（最優先）
2. デフォルト値・フォールバック表示
3. 画面遷移・連動制御

### `response_params[].note`（AA列インライン条件）
response_paramのAA列は「その行の表示条件」のみ記載する。
複数のAPIブロックにまたがる条件は`notes`（末尾注記）に移す。

### `display_mode`の判定
- `■ブロック表示`: 1件のデータを詳細表示する場合
- `■一覧表示`: 複数件をテーブルで表示する場合
- 両方ある場合は `display_mode` を `■ブロック表示` にして `response_params` の後に一覧表示セクションを別アクション（action_number付き）で定義する

## 7.4 コードからの逆引きチェックリスト

コード（Vue/APIエンドポイント/Spring Boot等）を渡された場合の抽出項目:

| コードの要素 | Configの対応フィールド |
|-------------|----------------------|
| コンポーネント名 + 物理名 | `processing.component_name` |
| onMounted / created / setup() のAPI呼び出し | `apis[0]` (action_type: 初期表示) |
| ボタン/アイコンのクリックハンドラー | `apis[n]` (action_number: n) |
| API エンドポイントパス | `apis[n].name` |
| composable引数 / request body のキー | `request_params[].name` |
| レスポンスの .then() での ref代入 | `response_params[].screen_item` + `response_param` |
| v-if / v-show / 条件分岐 | `notes` または `response_params[].note` |
