# 品質基準（完璧な設計書の条件）

## レベル1: 最低限（実装開始可能）
- [ ] 全UI要素に種類・動作が定義されている
- [ ] インタラクティブ要素に表示制御がある
- [ ] API連携がある操作にAPI IDが明記されている
- [ ] 処理詳細にリクエスト/レスポンスのマッピングがある

## レベル2: 標準（手戻りが少ない）
- [ ] エラーが起こりうる全操作にエラー定義がある
- [ ] 表示制御の条件が網羅的（モード/データ/権限/人数）
- [ ] 処理詳細のパラメーターにデータ型・形式の示唆がある
- [ ] 画面レイアウトと処理詳細の対応が明確

## レベル3: 理想（認識齟齬ゼロ）
- [ ] UI状態の定義がある（ローディング/空状態/エラー状態）
- [ ] エラー後の復帰動作が明記されている
- [ ] API レスポンスの条件分岐（null時、フラグ値による表示切替等）が詳細
- [ ] 境界条件が明記されている（最大件数、文字数制限等）
